## 1. 티켓 접수 및 처리

### Q1: 고객(Customer)은 어떤 방식으로 문의 티켓을 등록하나요?
- **A: 웹 포털 전용**
- 고객이 직접 웹사이트에 로그인하여 티켓을 작성합니다

### Q2: 티켓에 우선순위(Priority) 레벨이 필요한가요?
- **A: 3단계 (Low/Medium/High)**
- 기본적인 3단계 우선순위 체계를 사용합니다

### Q3: 티켓 제목 외에 필수 입력 항목이 있나요?
- **A: 제목 + 내용만**
- 제목과 문의 내용만 필수입니다

### Q4: 티켓 카테고리는 몇 단계로 구성하나요?
- **A: 1단계 (단순 목록)**
- 예: 결제, 배송, 반품 등 평면 목록 형태

### Q5: 고객이 티켓에 추가 댓글/정보를 남길 수 있나요?
- **A: 허용 (양방향 대화)**
- 고객이 티켓에 추가 댓글을 남길 수 있음

### Q6: 티켓 병합/분리 기능이 필요한가요?
- **A: 불필요**
- 병합/분리 없이 개별 처리

### Q7: 티켓 상태 변경 이력을 추적해야 하나요?
- **A: 전체 이력 추적**
- 상태, 담당자, 내용 변경 등 모든 이력 저장

---

## 2. 파일 첨부

### Q8: 티켓에 파일 첨부 기능이 필요한가요?
- **A: 이미지 + 문서**
- 이미지와 문서 파일(pdf, docx 등) 모두 허용

### Q9: 첨부파일 크기 제한은 얼마인가요?
- **A: 파일당 5MB**
- 개별 파일 최대 5MB

---

## 3. 담당자 할당 및 팀

### Q10: 담당자(Agent) 할당은 어떻게 이루어지나요?
- **A: 자동 할당 (Round-robin)**
- 시스템이 순차적으로 자동 배분

### Q11: Round-robin 할당 시 Agent 근무 상태를 고려하나요?
- **A: 온라인/오프라인 상태 고려**
- 온라인 Agent에게만 할당

### Q12: 티켓 담당자가 부재(휴가/퇴사) 시 어떻게 처리하나요?
- **A: 자동 재할당**
- 부재 설정 시 해당 티켓이 다른 Agent에게 자동 재배정

### Q13: 팀 구성은 어떻게 되나요?
- **A: 단일 팀**
- 모든 Agent가 하나의 팀에 소속

### Q14: Agent 한 명당 동시 처리 가능한 티켓 수에 제한이 있나요?
- **A: 제한 없음**
- Agent가 원하는 만큼 티켓 처리

### Q15: Agent끼리 내부 메모/노트를 남길 수 있나요? (고객에게 비공개)
- **A: 필요함**
- Agent간 비공개 노트 기능 제공

---

## 4. SLA (서비스 수준 협약)

### Q16: SLA(서비스 수준 협약) 관리가 필요한가요?
- **A: 기본 SLA (응답/해결 시간)**
- 우선순위별 응답 시간과 해결 시간 목표 설정

### Q17: 기본 SLA 기준 응답 시간은 얼마인가요? (첫 답변까지)
- **A: 1시간**
- 모든 티켓에 1시간 내 첫 응답

### Q18: 기본 SLA 기준 해결 시간은 얼마인가요? (티켓 해결까지)
- **A: 24시간**
- 모든 티켓 24시간 내 해결

---

## 5. 답변 전달 및 알림

### Q19: 고객에게 답변은 어떤 방식으로 전달되나요?
- **A: 이메일 전송**
- 답변을 고객 이메일로 발송

### Q20: Agent/Manager에게 실시간 알림이 필요한가요?
- **A: 이메일 알림만**
- 새 티켓, SLA 경고 등을 이메일로 알림

---

## 6. 티켓 재오픈

### Q21: 고객이 Closed 티켓을 다시 열 수 있나요?
- **A: 가능 (일정 기간 내)**
- Closed 후 일정 기간 내 같은 문제로 재오픈 가능

### Q22: Closed 티켓 재오픈 가능 기간은 얼마인가요?
- **A: 3일**
- Closed 후 3일 내 재오픈 가능

---

## 7. 고객 관련

### Q23: 고객 계정 등록은 어떻게 이루어지나요?
- **A: 자유 회원가입**
- 고객이 직접 계정을 등록

### Q24: 한 고객이 동시에 생성할 수 있는 티켓 수에 제한이 있나요?
- **A: 제한 없음**
- 고객이 원하는 만큼 티켓 생성 가능

### Q25: 해결된 티켓에 대해 고객 만족도 평가를 받나요?
- **A: 받음 (5점 척도)**
- 해결 후 1~5점 만족도 평가 요청

---

## 8. AI 기능

### Q26: AI 답변 제안 승인 프로세스는 어떻게 되나요?
- **A: Agent가 직접 수정/승인**
- Agent가 AI 초안을 확인하고 수정 후 바로 전송

### Q27: AI가 답변을 생성하지 못하는 경우 어떻게 처리하나요?
- **A: Agent가 직접 작성**
- AI 실패 시 Agent가 처음부터 직접 답변 작성

### Q28: AI 프롬프트 템플릿은 어떻게 관리되나요?
- **A: 카테고리별 템플릿**
- 각 카테고리마다 다른 AI 프롬프트 사용

### Q29: 감정 분석 결과는 어떻게 활용되나요?
- **A: 표시만 (참고용)**
- 부정/중립/긍정 표시, Agent 참고용

---

## 9. 지식베이스/FAQ

### Q30: 지식베이스/FAQ는 누가 접근할 수 있나요?
- **A: 내부용 (Agent/Manager만)**
- AI 답변 생성을 위한 내부 참고 자료로만 사용

---

## 10. 대시보드 및 보고서

### Q31: 대시보드에 표시할 핵심 지표는 무엇인가요?
- **A: 기본 지표 (티켓 수, 상태별 현황)**
- 열린/진행중/해결된 티켓 수 표시

### Q32: 통계/보고서는 어떤 형태로 제공하나요?
- **A: 화면 조회만**
- 웹 화면에서 통계 확인

### Q33: 티켓 검색 기능은 어느 수준으로 필요한가요?
- **A: 기본 필터링**
- 상태, 담당자, 날짜 등 기본 필터

---

## 11. 시스템 설정

### Q34: 사용자 인증 방식은 무엇인가요?
- **A: 이메일/비밀번호**
- 기본적인 이메일+비밀번호 로그인

### Q35: 다국어 지원이 필요한가요?
- **A: 한국어만**
- 한국어 단일 언어 지원

### Q36: 티켓 데이터 보존 기간은 어떻게 되나요?
- **A: 영구 보존**
- 모든 티켓 데이터를 영구적으로 보관

### Q37: 티켓 내용에 민감 정보(개인정보 등)가 포함될 수 있나요?
- **A: 포함됨 (마스킹 불필요)**
- 민감정보가 있으나 특별 처리 없이 저장

### Q38: Back Office와 Main App은 별도 URL/로그인인가요?
- **A: 통합 (권한으로 메뉴 구분)**
- 동일 URL, 사용자 권한에 따라 메뉴 표시

### Q39: 모바일 대응이 필요한가요?
- **A: 반응형 웹 (모바일 최적화)**
- 데스크톱/모바일 모두 지원하는 반응형 디자인
