 Phase 2 기능 테스트 가이드 ✅                                                                                 
                                                                                                                  로그인에 성공하셨네요! 이제 구현된 모든 기능을 테스트해보겠습니다.                                               
  1️⃣ 인증 시스템 테스트                                                                                         ─
  A. Admin 로그인 상태 확인

  - 우측 상단에 "관리자" 이름과 아바타 표시되는지 확인
  - 사이드바에 역할별 메뉴가 보이는지 확인
    - Dashboard
    - Tickets (아직 미구현)
    - Reports (아직 미구현)
    - Users (Admin만 보임)
    - Profile

  B. 사용자 메뉴 테스트

  1. 우측 상단 아바타 클릭
  2. 드롭다운 메뉴 확인:
    - 이름과 이메일 표시
    - "프로필" 버튼
    - "로그아웃" 버튼

  2️⃣ 사용자 관리 테스트 (Admin 기능)                                                                           


  URL: http://localhost:3002/users

  A. 사용자 목록 확인

  - 현재 Admin 계정(1명) 표시되는지 확인
  - 테이블 컬럼 확인:
    - 이름
    - 이메일
    - 역할 (배지)
    - 상태 (온라인/부재/오프라인)
    - 가입일
    - 작업 버튼

  B. 새 사용자 생성

  1. "사용자 추가" 버튼 클릭
  2. 다이얼로그에서 입력:
    - 이름: 테스트 에이전트
    - 이메일: agent@test.com
    - 비밀번호: Test123!
    - 역할: Agent 선택
  3. "저장" 클릭
  4. 목록에 새 사용자 추가되었는지 확인

  C. Agent 추가로 2명 더 생성

  에이전트 2:
  - 이름: 김담당
  - 이메일: kim@test.com
  - 비밀번호: Test123!
  - 역할: Agent

  에이전트 3:
  - 이름: 박매니저
  - 이메일: park@test.com
  - 비밀번호: Test123!
  - 역할: Manager

  D. Customer 계정 생성

  - 이름: 홍고객
  - 이메일: customer@test.com
  - 비밀번호: Test123!
  - 역할: Customer

  ✅ 확인: 총 4명의 사용자가 목록에 표시되어야 합니다.

  E. 검색 기능 테스트

  1. 상단 검색창에 "김" 입력
  2. "김담당"만 필터링되는지 확인
  3. 검색어 삭제하여 전체 목록 복구

  F. 역할 필터 테스트

  1. 역할 필터에서 "Agent" 선택
  2. Agent 역할 사용자만 표시되는지 확인
  3. "전체" 선택하여 복구

  G. Agent 상태 토글 테스트

  1. "테스트 에이전트" 행에서 상태 토글 버튼 클릭
  2. 온라인 ↔ 부재 상태 전환 확인
  3. 배지 색상 변경 확인:
    - 온라인: 초록색
    - 부재: 노란색
    - 오프라인: 회색

  H. 사용자 수정

  1. "김담당" 행에서 "수정" 버튼 클릭
  2. 이름을 "김에이전트"로 변경
  3. "저장" 클릭
  4. 목록에서 이름 변경 확인

  I. 사용자 삭제

  1. "홍고객" 행에서 "삭제" 버튼 클릭
  2. 확인 다이얼로그에서 "삭제" 클릭
  3. 목록에서 사용자 제거 확인

  3️⃣ 프로필 관리 테스트                                                                                        


  URL: http://localhost:3002/profile

  A. 프로필 조회

  - 현재 이름 표시: "관리자"
  - 현재 이메일 표시: admin@example.com (읽기 전용)
  - 역할 표시: Admin (읽기 전용)

  B. 이름 변경

  1. 이름을 "최고관리자"로 변경
  2. "저장" 클릭
  3. 성공 Toast 메시지 확인
  4. 우측 상단 헤더에서 이름 변경 확인

  C. 비밀번호 변경

  1. 현재 비밀번호: Admin123!
  2. 새 비밀번호: NewPass123!
  3. 비밀번호 확인: NewPass123!
  4. "비밀번호 변경" 클릭
  5. 성공 메시지 확인

  4️⃣ 회원가입 테스트                                                                                           


  A. 로그아웃

  1. 우측 상단 아바타 클릭
  2. "로그아웃" 클릭
  3. 로그인 페이지로 리다이렉트 확인

  B. 회원가입

  URL: http://localhost:3002/register

  1. 입력:
    - 이름: 신규고객
    - 이메일: newuser@test.com
    - 비밀번호: Test123!
    - 비밀번호 확인: Test123!
  2. "회원가입" 클릭
  3. 성공 메시지 및 로그인 페이지로 리다이렉트 확인

  C. 신규 계정으로 로그인

  1. 이메일: newuser@test.com
  2. 비밀번호: Test123!
  3. "로그인" 클릭
  4. 대시보드로 리다이렉트 확인

  5️⃣ 역할 기반 접근 제어 테스트                                                                                


  A. Customer 권한 테스트

  현재 로그인: newuser@test.com (Customer)

  1. 사이드바 확인:
    - "Users" 메뉴가 보이지 않아야 함
    - "Reports" 메뉴가 보이지 않아야 함
  2. URL 직접 접근 시도:
    - http://localhost:3002/users 입력
    - /dashboard로 자동 리다이렉트 확인

  B. Agent 권한 테스트

  1. 로그아웃
  2. agent@test.com / Test123! 로 로그인
  3. 사이드바 확인:
    - "Users" 메뉴가 보이지 않아야 함
    - "Reports" 메뉴가 보이지 않아야 함

  C. Manager 권한 테스트

  1. 로그아웃
  2. park@test.com / Test123! 로 로그인
  3. 사이드바 확인:
    - "Users" 메뉴가 보이지 않아야 함
    - "Reports" 메뉴는 보여야 함 (Phase 3에서 구현)

  D. Admin 권한 복구

  1. 로그아웃
  2. admin@example.com / NewPass123! (변경한 비밀번호) 로 로그인
  3. 모든 메뉴 접근 가능 확인

  6️⃣ 세션 관리 테스트                                                                                          


  A. 세션 유지 확인

  1. 로그인 상태에서 브라우저 새로고침 (F5)
  2. 로그인 상태 유지되는지 확인
  3. 사용자 정보가 계속 표시되는지 확인

  B. 보호된 페이지 접근

  1. 새 시크릿/프라이빗 창 열기
  2. http://localhost:3002/dashboard 직접 접근
  3. 로그인 페이지로 리다이렉트 확인
  4. URL에 ?callbackUrl=/dashboard 파라미터 있는지 확인

  7️⃣ 반응형 디자인 테스트                                                                                      


  A. 모바일 뷰 (F12 → 모바일 시뮬레이션)

  1. 개발자 도구 열기 (F12)
  2. 모바일 디바이스 시뮬레이션 활성화
  3. 사이드바가 햄버거 메뉴로 변경되는지 확인
  4. 테이블이 반응형으로 조정되는지 확인
