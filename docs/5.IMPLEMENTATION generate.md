Create specs/IMPLEMENTATION.md:

Phase-by-phase implementation plan with checkboxes:

Each Phase include:
1. Goal
2. Estimated time
3. Files to create (checklist format)
4. Implementation strategy
5. Testing checkpoint (checklist)
6. Acceptance criteria (checklist)

Phase structure:
- Phase 1: Project Setup & PDF Upload UI
- Phase 2: OpenRouter API Integration & Analysis
- Phase 3: Results Display & Risk Visualization
- Phase 4: Advanced Features & Polish

Example format:
```
## Phase 1: Basic UI (1 hour)

**Goal:** Create PDF upload interface

**Estimated Time:** 1 hour

**Files to Create:**
- [ ] lib/types.ts
- [ ] components/UploadForm.tsx
- [ ] components/ContractViewer.tsx
- [ ] app/page.tsx

**Implementation Strategy:**
types.ts first (other files depend on it)
UploadForm and ContractViewer can be built in parallel
page.tsx last (imports all components)

**Testing Checklist:**
- [ ] npm run dev runs without errors
- [ ] Upload form displays correctly
- [ ] File upload works
- [ ] PDF preview shows
- [ ] Mobile responsive

**Acceptance Criteria:**
- [ ] Upload functional
- [ ] PDF displays correctly
- [ ] Clean UI
- [ ] No console errors
```

Use checkbox format (- [ ]) for all lists.
Write in natural language.
Keep it simple and actionable.
