SRS.md 파일을 읽어줘. 이 SRS는 불완전하다고 가정해.

고객이 명시하지 않았지만 반드시 결정해야 할 요구사항들을 찾아내기 위해 인터뷰를 진행해줘.

중요: 모든 질문에 반드시 다음 형식으로 선택지를 구성해줘:
1. [구체적 옵션 A]
2. [구체적 옵션 B]
3. [구체적 옵션 C]
4. 이 기능 제외 (Skip)
5. Type something

사용자가 "이 기능 제외 (Skip)"을 선택하면 해당 주제의 모든 후속 질문을 건너뛰고 다음 주제로 즉시 이동해줘.

다음 관점에서 질문해줘:
- SRS에 명시되지 않은 기능 요구사항
- 숨겨진 사용자 니즈와 기대사항
- 고려되지 않은 사용 시나리오
- 요구사항 간의 모순이나 충돌
- 빠진 제약사항이나 한계
- 명확하지 않은 우선순위
- 예외 상황과 엣지 케이스

한 번에 3-5개 질문만 하고, 내 답변에 따라 후속 질문을 생성해줘.
기술적 구현 방법은 묻지 말고, "무엇을" 만들어야 하는지에 집중해줘.

모든 요구사항이 논리적으로 완전해질 때까지 계속 질문해줘.

인터뷰가 완료되면:
1. 모든 질문과 답변을 /docs/	INTERVIEW.md 파일로 저장해줘.
2. INTERVIEW.md 저장 후 즉시 멈춰줘. 구현하지 마.
3. 다음 단계는 내가 별도로 지시할 때까지 대기해줘.
